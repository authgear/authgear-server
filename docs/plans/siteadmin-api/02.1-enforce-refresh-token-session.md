# 02.1 — Enforce refresh_token Session Type for Site Admin API

## Context

`SessionInfoMiddleware` supports two authentication modes, selected by
`AuthgearConfig.WebSDKSessionType` (env var `WEB_SDK_SESSION_TYPE`, default `"cookie"`):

| Mode | Mechanism | Used by |
|---|---|---|
| `refresh_token` | Bearer JWT in `Authorization` header; validates against JWKS endpoint | Site admin REST clients |
| `cookie` | Parses X-Authgear-* request headers set by a cookie proxy | Portal web UI |

The site admin API is called by a web client that authenticates using the Authgear Web SDK in
`refresh_token` mode — the SDK holds a refresh token, exchanges it for an access token, and
sends that access token as a Bearer JWT in the `Authorization` header.

The `cookie` mode is designed for the portal web UI, where a cookie proxy sets X-Authgear-*
request headers that the middleware trusts. That proxy-header pattern is not part of the site
admin web client's authentication flow.

However, the current DI wiring passes the shared `AuthgearConfig` directly into
`SessionInfoMiddleware`, so the mode is controlled by `WEB_SDK_SESSION_TYPE` (default
`"cookie"`). If the env var is absent or misconfigured, siteadmin silently falls back to
`cookie` mode and the web client's Bearer tokens are ignored — requests are treated as
unauthenticated or authenticated via the wrong mechanism.

**Goal:** make `refresh_token` an unconfigurable constant for the site admin service, enforced
in DI. Operators cannot accidentally (or deliberately) switch it to `cookie`.

---

## Current wiring

`pkg/siteadmin/deps.go` includes `session.DependencySet`, which provides
`*session.SessionInfoMiddleware` via `wire.Struct(new(SessionInfoMiddleware), "*")`. This
injects `rootProvider.AuthgearConfig` directly, so `WebSDKSessionType` is read from the env var.

```
RootProvider.AuthgearConfig  ──►  SessionInfoMiddleware.AuthgearConfig
                                  (WebSDKSessionType read at request time)
```

---

## Fix

Replace `session.DependencySet` in `pkg/siteadmin/deps.go` with a custom provider that copies
`AuthgearConfig` and unconditionally sets `WebSDKSessionType = "refresh_token"` before
constructing `SessionInfoMiddleware`. This change is local to the siteadmin package; no shared
code is modified.

```
RootProvider.AuthgearConfig  ──►  provideSiteAdminSessionInfoMiddleware
                                  (copies config, forces WebSDKSessionType = "refresh_token")
                                  ──►  SessionInfoMiddleware
```

---

## Implementation

### `pkg/siteadmin/deps.go`

Remove `session.DependencySet` and replace with individual providers:

```go
// Remove:
session.DependencySet,

// Add:
session.NewHTTPClient,
provideSiteAdminSessionInfoMiddleware,
```

Add the provider function (also in `deps.go`):

```go
// provideSiteAdminSessionInfoMiddleware constructs SessionInfoMiddleware with
// WebSDKSessionType hard-wired to "refresh_token". The site admin web client
// uses the Authgear Web SDK in refresh_token mode and authenticates via Bearer
// JWT in the Authorization header. Cookie mode (the portal default) is not
// part of the site admin authentication flow and must never be used here.
func provideSiteAdminSessionInfoMiddleware(
    c *portalconfig.AuthgearConfig,
    httpClient session.HTTPClient,
    clock clock.Clock,
) *session.SessionInfoMiddleware {
    cfg := *c                              // shallow copy — safe for string fields
    cfg.WebSDKSessionType = "refresh_token"
    return &session.SessionInfoMiddleware{
        AuthgearConfig: &cfg,
        HTTPClient:     httpClient,
        Clock:          clock,
    }
}
```

No changes to `pkg/siteadmin/wire.go` or `routes.go` — the binding
`wire.Bind(new(httproute.Middleware), new(*session.SessionInfoMiddleware))` remains unchanged.
